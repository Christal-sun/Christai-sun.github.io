学习日记(三)

发表于2024-04-25|更新于2024-05-10|JavaSE学习

一、包装类

  • 什么是包装类

    Java把8种基本类型,进行了封装,变成包装类

    基本类型 包装类
    byte Byte
    short Short
    int Integer
    long Long
    float Float
    double Double
    char Character
    boolean Boolean

  • 包装类解决程序中的什么问题

    1. 实现String类型的7种基本类型(没有char)之间的转换

    2. 在学习mysql时会大量用到包装类(包装类中可以存储null数据)

  • 包装类怎么使用

  • 包装类在使用中的细节

    1. 在把String类型数据转换为基本类型数据时,容易发生异常

      • 原因:String类型数据,不符合要转换的基本类型数据格式

    2. 包装类在使用时存在: 自动装箱,自动拆箱

      • 自动装箱: Integer.valueOf(数值)
      • 自动拆箱: Integer对象.intValue()

    3. 包装内中有常量池(数据值没有超出常量池范围,就直接从常量池中获取对象)

二、正则表达式

  • 什么是正则表达式

    由一些特定的字符组成的字符串校验规则

  • 正则表达式解决程序中的什么问题

    • 只能针对字符串进行格式校验

    • 在开发中的应用场景:对用户输入的字符串数据进行校验(数据合法性,例如: 手机号码格式)

三、Collection集合

集合不能存储基本类型数据,集合有三种:List,Set,Map

  • 集合有什么好处

    1. 大小可变(随意扩容)
    2. 集合中可以存储多种不同类型数据
    3. 底层使用数据结构(存取效率高)

  • Collection集合是什么

    Collection集合是一个接口

    • 子接口:List,Set
    • 子类:ArrayList

  • ArrayList集合对象可以使用传统的for循环，进行遍历

    • ArrayList集合有索引

  • Collection集合： 没有索引 . 遍历集合使用专用方式：迭代器

四、迭代器

  • 用于集合的遍历

  • Iterator迭代器中常用方法

  • 迭代器使用注意事项

    1. 在迭代器完成集合的遍历后,不能再使用next()方法
    2. 在迭代器遍历过程中,不能使用集合对象来增删元素
      • 删除元素: 使用迭代器对象中的remove方法实现
      • 增加元素: 不用考虑(解决方案: 使用其他的集合对象.例: List集合)

五、增强for循环(foreach)

  • 基于普通的for循环,进行强化(底层使用:迭代器)

  • 应用

    • 针对数组或集合,进行遍历

  • 语法

六、泛型

  • 泛型是什么

    泛型是一种类型参数,专门用来保存类型.

  • 泛型解决程序中什么问题

    1. 在创建集合对象时,明确了集合中所存储元素的类型(限定类型)
    2. 泛型是使用在代码编写时期的技术方式(编译器技术)
    3. 泛型在程序运行后就擦除

  • 泛型的使用

    • 泛型类

    • 泛型接口

    • 泛型方法

  • 开发时选择哪个泛型定义

    1. 先明确要定义的是接口还是类
      • 当类中的成员变量在不确定具体使用哪种类型时,可以使用: 泛型,表示类型
      • 当接口中方法在不确定方法参数类型或方法返回值类型时,可以使用: 泛型,表示类型
    2. 在编写工具类时,工具类中的方法参数或方法返回值不确定具体类型,可以使用: 泛型

  • 泛型在使用中的注意细节

    • 泛型不支持多态

    • 泛型在语法上支持通配符 ? //表示任意类型

    • 泛型中的通配符,通常会配置上下限使用

七、List,Set集合(单列集合)

  • Java语言中集合体系的划分:

    • Collection (接口)
    • Map (接口)

  • java.util.Collection集合: 是一个接口(无法实例化)

  • java.util.List集合 (接口)

    • 常用子类:ArrayList, LinkedList

    • 带有索引

    • 集合中存储元素的顺序和获取元素的顺序一致

    • 可以存储重复元素

    • 因为List集合可以使用索引,故围绕着索引,可以设计很多API方法

  • ArrayList集合

    • 实现List接口(List接口中的所有功能都有)
    • 底层使用: 数组
      • 查询快,增删慢

  • LinkedList集合

    • 实现List接口

    • 底层使用: 链表

      • 增删快,查询慢

    • 特有方法都围绕着链表头和尾设计

  • java.util.Set集合 (接口)

    • 没有索引

    • 存取元素不保证顺序

    • 不允许存储重复元素

    • Set集合方法全部来自Collection集合

    • 常用子类:

      • HashSet集合

        • 特点:
          1. 没有索引
          2. 不允许存储重复元素
          3. 存取元素不保证顺序
        • 底层使用: 哈希表结构 (哈希表去重原理: 存储的元素必须重写hashC哦的,equals方法: )
        • JDK1.8底层优化了哈斯表(当链表长度超出8,就自动转为红黑树)

      • LinkedHashSet集合

        • 特点:
          1. 没有索引
          2. 不允许存储重复元素
          3. 存取元素顺序一致
        • 底层使用:哈希表 + 链表(保证存储元素的顺序)

      • TreeSet集合

        • 特点

          1. 不能存储重复元素
          2. 没有索引
          3. 存储的元素会按照规则进行排序

        • 底层使用: 树(红黑树)结构(去重,排序)

        • 在TreeSet集合中存储: String, Integer, Double, Character //JDK提供的类型

          • 默认已经实现了java.lang.Comparable接口(自带自然排序规则)

        • 在TreeSet集合中存储: 自定义类型

          • 必须保证自定义类型有实现java.lang.Comparable接口,并重写compateTo方法
          • 如果自定义类型没有实现Comparable接口,在存储到TreeSet集合中,会引发异常

八、Map集合(双列集合)

  • 特点

    1. 可以存储两个元素(键值对元素)
    2. key元素不能重复,value元素可以重复
    3. 一个key元素只能对应一个value元素(一一对应)//通过key可以找到value
    4. 存取元素不保证顺序
    5. 没有索引

  • HashMap

    • 底层使用：哈希表

  • LinkedHashMap

    • 底层使用： 哈希表 + 链表
    • 特点：
      1. 元素唯一
      2. 元素有序

  • TreeMap

    • 底层使用：红黑树
    • 特点：存储的key元素，会按照制定规则排序
      1. 自然排序：元素自身实现Comparable接口
      2. 比较器排序：在创建集合对象时，指定比较器

  • Map集合遍历

    Map集合不能直接遍历

    1. 键找值

      • 获取Map集合中所有的key元素，遍历所有的key元素，通过key元素找到对应的value元素

    2. 键值对

      • 获取Map集合中所有的Map.Entry，遍历所有的Map.Entry，通过Entry中的API方法获取到key、value

    在Map集合中当key存储的是自定义对象时，要保证对象存储数据的唯一性，需要： 自定义对象中重写hashCode、equals方法
